发表于 2017-05-26 | 分类于 redis | 本文总阅读量 次

在redis的src源码目录下，除了底层数据结构的实现，还有一类文件，比如t_string.c,t_list.c等，它的作用就是负责客户端与服务端的通信，server接受client的指令调用相应的对象命令函数解析后执行，对内存中redisobj操作后，返回给client相应的信息。所以这一类文件中都包含有5中数据对象的指令执行函数，即command api，对于复杂的数据结构，对redisobj中数据值（robj->ptr）的操作有一次进行了封装,即list api，hash api等等。
