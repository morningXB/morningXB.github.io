强化学习(7)——Approximation Function

发表于 2018-06-11 | 分类于 强化学习

Introduction

1.why we need Approximation Function？
当状态空间过大，或者状态空间连续时，难以用Q表格来解决问题，所以我们需要一个函数称为近似函数，传入当前状态以及动作计算得到Q的近似值，通过有限的状态动作对，我们拟合参数，从而计算未出现的状态动作对Q值。
2.what is the type of Approximation Function？
近似函数有线性和非线性之分，常用的类型有用线性回归拟合的线性函数和用反向传播拟合的人工神经网络。
3.the common learning method？
增量法即随机梯度下降法，注重在线学习，其次是批处理法即批梯度下降法，注重的是数据利用的高效性。
4.the converge of Approximation Function？
对于MC来说，线性非线性函数的预测都可以收敛，而TD在非线性上难以收敛，原因是MC总是提供奖励回馈的无偏估计，而TD采用带偏差的估计值。对于Control问题，他们在非线性函数都难以收敛，其他具体表现下面会列出。为了解决非线性函数不收敛的问题deepmind提出了DQN(Deep Q-learning Network)方法，将深度学习与强化学习结合，在之后的章节会专门讲讲DQN。

发表于 2018-05-30 | 分类于 强化学习

MC Control

回顾一下policy-iteration的control流程，分为两步：1.对当前policy进行评估获得值函数；2.贪婪策略进行策略优化。循环上述过程直到收敛至最优策略。
在Model-Free的前提下，MC-Control方法中我们在第一步采用MC-Prediction方法进行值函数评估，如此出现几个问题：
1.MC-Prediction后的获得状态值函数V，在我们模型未知的前提下,下一步无法优化模型，所以我们第一步评估policy的Q函数即状态-动作对函数。这样根据Q函数可以轻易选取当前状态最优动作以优化策略，Q的求取方法同V；
2.Model-Free的值评估采用多次取样逼近真值，并没有探索所有可能路径，所以采用Greedy优化策略会错过一些最佳路径，涉及强化学习的探索和利用问题，这里举一个常用解决方法，$\epsilon$-Greedy优化方法，如下。

强化学习(4)——Model-Free—Prediction

发表于 2018-05-30 | 分类于 强化学习

Model-Free Samples

按照policy_iteration的整体思想，control(优化策略)的前提时获得当前策略的值函数，所以我们先说Model-Free情况的Prediction问题(求值函数)。
在不知道model的情况下，没有办法通过bellman方程迭代预测当前policy的各状态的值，概率学中大数定理，也就对各状态下的值进行抽样，取抽样集的数学期望作为近似的值，当抽样集的数目无穷大时，我们获得理想的状态值，对数学期望的求取进行改进，改为差值递增逼近。

$\alpha$可以理解为我们熟悉的learning rate。$G_t$即当前时刻状态的奖励Return。通过不断采样带入计算，无限逼近值函数。
MC以及TD方法的区别就在于$G_t$的不同，MC(蒙特卡罗法)中取值是一个episode中真实的无偏差值，TD(时序差分法)方法则采用bellman方程的思想，取值为当前获得立即Reward加上后一个状态的值。两者的比较在下面细说。
